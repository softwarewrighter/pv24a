Reproduced and characterized TRAP 3 failure in linked programs. Root cause is two resource limits, not a VM opcode bug: (1) Symbol table overflow - full pr24p runtime has 93 symbols but pvmasm.s max is 64 entries, causing SYM? FULL errors and garbled bytecode. (2) UART input feed rate - at speed=0, cor24-run feeds ~1 char per 10K instructions, so 12KB+ inputs need 127M+ instructions just to read. Small/medium linked programs (3-21 procs, <64 symbols) work correctly with sufficient instruction budget. Created test cases: merged_int.spc (3 procs), merged_str.spc (3 procs with .data), merged_mid_int.spc (8 procs), merged_noenter.spc (no enter/leave), merged_21procs.spc (21 procs/59 symbols). All produce correct output. Also fixed REQ-026 (LED address 0xFF0200->0xFF0000), REQ-025 (sys 6 read_switch in pvm.s+pvmasm.s), and active-low LED inversion.
